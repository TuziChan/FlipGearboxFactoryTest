## 1. 系统概述

齿轮箱测试系统是一套自动化测试平台，用于对齿轮箱产品进行质量检测。系统通过Modbus RTU协议控制电机驱动器、扭矩传感器、编码器和制动电源，执行标准化测试流程。

### 1.1 主要功能
- 自动化测试流程执行
- 实时数据采集（33ms周期）
- 测试结果判定
- 测试报告生成
- 历史记录查询
- 配方管理

## 2. 系统启动

### 2.1 启动前检查
1. 确认所有硬件设备已正确连接
2. 检查串口连接状态
3. 确认配置文件存在：`config/station.json`
4. 确认配方文件存在：`config/recipes/*.json`

### 2.2 启动方式

#### 模拟模式（无硬件）
```bash
appFlipGearboxFactoryTest.exe --mock
```

#### 真实硬件模式
```bash
appFlipGearboxFactoryTest.exe
```

### 2.3 启动日志
系统启动后会在`logs/`目录下生成日志文件：
- 文件名格式：`app_YYYY-MM-DD.log`
- 日志自动按天轮转
- 保留最近30天日志

## 3. 界面操作

### 3.1 主界面布局
- **顶部工具栏**：系统状态、模式切换
- **左侧导航栏**：功能页面切换
- **中央工作区**：当前页面内容
- **底部状态栏**：实时状态信息

### 3.2 测试执行页面

#### 3.2.1 开始测试
1. 在配方选择下拉框中选择测试配方
2. 输入齿轮箱序列号
3. 点击"开始测试"按钮
4. 系统自动执行测试流程

#### 3.2.2 测试流程
测试流程包含以下阶段：
1. **归零阶段**：电机寻找编码器零点
2. **空载阶段**：短暂停留，准备加载
3. **角度阶段**：旋转指定角度，检测扭矩
4. **负载阶段**：施加制动负载，检测性能

#### 3.2.3 实时监控
- **实时曲线**：扭矩、速度、角度、电流
- **当前状态**：显示当前测试阶段
- **进度条**：显示测试进度
- **数据表格**：实时遥测数据

#### 3.2.4 测试结果
测试完成后显示：
- **判定结果**：PASS/FAIL
- **失败原因**：如果失败，显示具体原因
- **性能指标**：最大/平均扭矩、速度、电流等
- **测试时长**：总测试时间

## 4. 配方管理

### 4.1 查看配方
1. 点击左侧导航栏"配方管理"
2. 配方列表显示所有可用配方
3. 点击配方查看详细参数

### 4.2 创建配方
1. 点击"新建配方"按钮
2. 填写配方基本信息：
   - 配方名称
   - 齿轮箱型号
   - 描述信息
3. 配置各阶段参数：
   - 归零阶段：目标速度、最大时长
   - 空载阶段：停留时长
   - 角度阶段：目标角度、速度、扭矩范围
   - 负载阶段：目标速度、制动电压、扭矩范围、电流限制
4. 点击"保存"按钮

### 4.3 编辑配方
1. 在配方列表中选择配方
2. 点击"编辑"按钮
3. 修改参数后点击"保存"

### 4.4 删除配方
1. 在配方列表中选择配方
2. 点击"删除"按钮
3. 确认删除操作

## 7. 配置管理

### 7.1 站点配置
配置文件：`config/station.json`

主要参数：
- `stationId`: 工站ID
- `stationName`: 工站名称
- `brakeChannel`: 制动通道号
- `defaultRecipe`: 默认配方

### 7.2 设备配置
每个设备包含以下参数：
- `portName`: 串口名称（COM1-COM256）
- `slaveId`: Modbus从站地址（1-247）
- `baudRate`: 波特率（9600/19200/38400/57600/115200）
- `timeout`: 超时时间（100-5000ms）
- `parity`: 校验位（None/Even/Odd）
- `stopBits`: 停止位（1/2）
- `enabled`: 是否启用

## 8. 故障排查

### 8.1 系统无法启动
**症状**：双击程序无响应

**解决方法**：
1. 检查Qt运行库是否安装
2. 查看日志文件：`logs/app_YYYY-MM-DD.log`
3. 确认配置文件格式正确

### 8.2 设备连接失败
**症状**：设备状态显示红色

**解决方法**：
1. 检查串口连接
2. 确认串口号配置正确
3. 检查设备电源
4. 使用串口调试工具测试通信

### 8.3 测试失败
**症状**：测试结果显示FAIL

**常见原因**：
- 扭矩超出范围：检查齿轮箱质量
- 速度不稳定：检查电机驱动器
- 角度偏差过大：检查编码器零点
- 电流过大：检查制动电源设置

### 8.4 数据采集异常
**症状**：实时曲线不更新

**解决方法**：
1. 检查采集调度器状态
2. 查看日志中的错误信息
3. 重启系统
