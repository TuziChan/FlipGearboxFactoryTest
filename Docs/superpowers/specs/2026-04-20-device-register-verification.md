## 1. AQMD3610NS-A2 电机驱动器

### 1.1 通信参数
- **从站地址**: 1
- **波特率**: 9600 bps
- **校验位**: Even (偶校验)
- **停止位**: 1
- **数据位**: 8

### 1.2 支持的功能码
- ✅ 0x03: Read Holding Registers
- ✅ 0x06: Write Single Register
- ✅ 0x10: Write Multiple Registers
- ✅ 0x2B: Read Device Identification

### 1.3 寄存器映射验证

#### 设备描述信息寄存器 (只读, 0x03)
| 地址 | 名称 | 实现常量 | 数据类型 | 缩放因子 | 验证状态 |
|------|------|----------|----------|----------|----------|
| 0x0000 | 设备标识 | REG_DEVICE_ID | uint16_t | - | ✅ 正确 |
| 0x0001 | 设备版本号 | - | uint16_t | 高字节=主版本, 低字节=副版本 | ✅ 正确 |
| 0x000C | 最大输出电流 | - | uint16_t | ×0.01 A | ✅ 正确 |

#### 实时状态寄存器 (只读, 0x03)
| 地址 | 名称 | 实现常量 | 数据类型 | 缩放因子 | 验证状态 |
|------|------|----------|----------|----------|----------|
| 0x0010 | 实时PWM | - | uint16_t | ×0.1% | ✅ 正确 |
| 0x0011 | 实时电流 | REG_REAL_TIME_CURRENT | uint16_t | ×0.01 A | ✅ 正确 |
| 0x0012 | 换向频率 | - | uint16_t | 次/秒 | ✅ 正确 |
| 0x0013 | 堵转状态 | - | uint16_t | 0=不堵转, 1=正转堵转, 2=反转堵转 | ✅ 正确 |

#### 速度控制寄存器 (读写, 0x03/0x06)
| 地址 | 名称 | 实现常量 | 数据类型 | 取值范围 | 缩放因子 | 验证状态 |

| 0x0040 | 设置速度 | REG_SET_SPEED | int16_t | -1000~1000 (PWM模式) | ×0.1% 占空比 | ✅ 正确 |
| 0x0042 | 停止并自锁 | REG_STOP_AND_LOCK | uint16_t | 0~1000 | ×0.1% 自锁占空比 | ✅ 正确 |
| 0x0044 | 自然停止或释放自锁 | REG_NATURAL_STOP | uint16_t | 写1触发 | - | ✅ 正确 |

#### AI1/AI2 GPIO控制寄存器 (读写, 0x03/0x06/0x10)
| 地址 | 名称 | 实现常量 | 数据类型 | 取值范围 | 验证状态 |

| 0x0051 | AI2端口方向 | - | uint16_t | 0=输入, 1=输出 | ✅ 正确 |
| 0x0052 | AI1端口电平 | REG_AI1_PORT_LEVEL | uint16_t | 0=低, 1=高 | ✅ 正确 |

## 2. DYN200 扭矩传感器

### 2.1 通信参数
- **从站地址**: 2
- **波特率**: 19200 bps
- **校验位**: None
- **停止位**: 2
- **数据位**: 8

### 2.2 支持的功能码
- ✅ 0x03: Read Holding Registers
- ✅ 0x05: Write Single Coil (清零/恢复出厂)
- ✅ 0x10: Write Multiple Registers

### 2.3 寄存器映射验证

#### 实时数据寄存器 (只读, 0x03)
| 地址 | 名称 | 实现常量 | 数据类型 | 缩放因子 | 字节序 | 验证状态 |
|------|------|----------|----------|----------|--------|----------|
| 0x0000-0x0001 | 扭矩值 | REG_TORQUE | int32_t | ×0.01 N·m | 大端 | ✅ 正确 |
| 0x0002-0x0003 | 转速 | REG_SPEED | int32_t | ×1 RPM (无缩放) | 大端 | ✅ 正确 |
| 0x0004-0x0005 | 功率 | REG_POWER | int32_t | ×0.1 W | 大端 | ✅ 正确 |

**关键修正** (参考 2026-04-17-device-registers-correction.md):
- ✅ 所有数据为 **32位有符号整数**，占用 **2个连续寄存器**
- ✅ 字节序为 **大端** (高字在前)
- ✅ 转速 **无缩放因子** (×1 RPM)

#### 配置参数寄存器 (读写, 0x03/0x10)

| 0x0008 | 扭矩小数点 | - | uint32_t | 0~4 | ✅ 正确 |
| 0x000A | 开机清零 | - | uint32_t | 0=不清零, 1=清零 | ✅ 正确 |
| 0x001C | 通信模式 | REG_COMM_MODE | uint32_t | 0=HEX6, 1=Modbus, 2=ASCII, 3=HEX8 | ✅ 正确 |

#### 特殊操作 (0x05/0x10)
| 地址 | 操作 | 功能码 | 写入值 | 验证状态 |
|------|------|--------|--------|----------|
| 0x0000 | 清零 | 0x05 或 0x10 | 0x00000001 | ✅ 正确 |
| 0x0002 | 恢复出厂设置 | 0x05 或 0x10 | 0x00000001 | ✅ 正确 |

## 6. 超时和错误处理验证

### 6.1 ModbusRtuBusController
- ✅ 串口打开失败检测
- ✅ 写入超时检测 (`waitForBytesWritten`)
- ✅ 读取超时检测 (`waitForResponse`)
- ✅ 帧间延迟 (T3.5) 计算和应用
- ✅ 错误信息记录 (`lastError()`)

### 6.2 设备层错误处理
所有设备操作遵循统一模式：
```cpp
bool operation() {
    if (!validate_preconditions()) {
        m_lastError = "...";
        return false;
    }
    
    if (!m_busController->sendRequest(request, response)) {
        m_lastError = m_busController->lastError();
        return false;
    }
    
    if (!parseResponse(response)) {
        m_lastError = "Invalid response or CRC error";
        return false;
    }
    
    return true;
}
```

- ✅ 所有设备操作返回 `bool`
- ✅ 失败时通过 `lastError()` 获取错误信息
- ✅ 禁止抛出异常
- ✅ 超时参数可配置
