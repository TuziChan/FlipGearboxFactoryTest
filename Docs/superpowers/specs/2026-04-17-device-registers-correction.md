## AQMD3610NS-A2 驱动器（修正版）

### 通信参数
- 默认站号：`0x01`（可配置 `0x01-0xF7`）
- 波特率：`9600` / `19200` / `38400`（通过拨码开关配置）
- 数据位：`8`
- 校验位：`无校验 + 1 停止位`（推荐）
- 超时：`500ms`

### 实时状态寄存器（只读，功能码 0x03）

| 地址 | 名称 | 数据类型 | 范围 | 缩放 | 说明 |
|------|------|----------|------|------|------|
| `0x0010` | 实时 PWM | uint16 | 0-1000 | ×0.1% | 当前输出占空比 |
| `0x0011` | 实时电流 | uint16 | 0-1000 | ×0.01A | 当前电机电流 |
| `0x0012` | 换向频率 | uint16 | 0-2000 | 次/秒 | 用于计算转速 |
| `0x0013` | 堵转状态 | uint16 | 0-2 | - | 0=正常, 1=正转堵转, 2=反转堵转 |
| `0x0014` | AI1 电压 | uint16 | 0-1000 | ×0.01V | AI1 模拟电压 |
| `0x0015` | AI2 电压 | uint16 | 0-1000 | ×0.01V | AI2 模拟电压 |
| `0x0018` | SQ1 电平 | uint16 | 0-1 | - | 0=低, 1=高 |
| `0x0019` | SQ2 电平 | uint16 | 0-1 | - | 0=低, 1=高 |

### 速度控制寄存器（读写，功能码 0x06）

| 地址 | 名称 | 数据类型 | 范围 | 缩放 | 说明 |
|------|------|----------|------|------|------|
| `0x0040` | 设置速度 | int16 | -1000~1000 | 见备注 | **有符号**，>0 正转，<0 反转，=0 刹车 |
| `0x0042` | 停止并自锁 | uint16 | 0-1000 | ×0.1% | 写入自锁占空比并停止 |
| `0x0044` | 自然停止 | uint16 | 1 | - | 写入 1 释放自锁 |

**0x0040 寄存器备注：**
- PWM 调速方式：数值 ×0.1% = 占空比
- 转矩调速方式：数值 ×0.01 = 电流(A)
- 自测速闭环方式：数值 = 换向频率(次/秒)
- 外接测速发电机方式：数值 ×0.01 = 反馈电压(V)

### AI1/AI2 作 GPIO 使用寄存器（功能码 0x03/0x06/0x10）

| 地址 | 名称 | 数据类型 | 范围 | 功能码 | 说明 |
|------|------|----------|------|--------|------|
| `0x0050` | AI1 端口方向 | uint16 | 0-1 | 0x03/0x06/0x10 | 0=输入, 1=输出 |
| `0x0051` | AI2 端口方向 | uint16 | 0-1 | 0x03/0x06/0x10 | 0=输入, 1=输出 |
| `0x0052` | AI1 端口电平 | uint16 | 0-1 | 0x03/0x06/0x10 | 0=低电平, 1=高电平 |
| `0x0053` | AI2 端口电平 | uint16 | 0-1 | 0x03/0x06/0x10 | 0=低电平, 1=高电平 |

**重要说明：**
- 本项目使用 AI1 作为磁铁感应器输入，需要先配置 `0x0050 = 0`（输入模式）
- 读取 `0x0052` 获取 AI1 电平，`0` 表示磁铁命中（低电平）
- 使用 0x10 功能码批量写 4 个寄存器时，会同时保存配置到 EEPROM

### 使用示例

**启动电机（PWM 模式，50% 占空比正转）：**
```
写 0x0040 = 500  (500 × 0.1% = 50%)
```

**读取电机电流：**
```
读 0x0011，结果 235 表示 2.35A
```

**读取 AI1 磁铁事件：**
```
1. 配置 AI1 为输入：写 0x0050 = 0
2. 读取 AI1 电平：读 0x0052
   - 结果 0 = 低电平（磁铁命中）
   - 结果 1 = 高电平（无磁铁）
```

## DYN200 扭矩传感器（修正版）

### 通信参数
- 默认站号：`0x02`
- 波特率：`9600` / `19200`（可配置）
- 数据位：`8`
- 校验位：`无校验 + 1 停止位`
- 超时：`300ms`
- **重要：** Modbus 输出频率上限约 30Hz

### 数据寄存器（只读，功能码 0x03）

| 起始地址 | 名称 | 数据类型 | 字节数 | 缩放 | 说明 |
|----------|------|----------|--------|------|------|
| `0x00` | 扭矩 | int32 | 4 | ×0.01 N·m | 有符号 32 位整数 |
| `0x02` | 转速 | int32 | 4 | ×1 RPM | 有符号 32 位整数 |
| `0x04` | 功率 | int32 | 4 | ×0.1 W | 有符号 32 位整数 |

**重要说明：**
- DYN200 每个数据占用 **2 个 Modbus 寄存器**（4 字节）
- 读取时需要使用功能码 0x03，读取 2 个连续寄存器
- 数据为有符号 32 位整数，需要按大端序（Big-Endian）解析

### 控制寄存器（读写，功能码 0x06）

| 地址 | 名称 | 数据类型 | 范围 | 功能码 | 说明 |
|------|------|----------|------|--------|------|
| `0x10` | 零点校准 | uint16 | 1 | 0x06 | 写入 1 执行零点校准 |

### 使用示例

**读取转速：**
```
读 0x0002，长度 2（读取 2 个寄存器，共 4 字节）
结果：[0x00, 0x00, 0x04, 0xD2] = 1250 RPM
```

**读取扭矩：**
```
读 0x0000，长度 2
结果：[0xFF, 0xFF, 0xFE, 0xA2] = -350 → -3.50 N·m
```

**零点校准：**
```
写 0x0010 = 1
```

## 单圈编码器（修正版）

### 通信参数
- 默认站号：`0x03`
- 波特率：`9600` / `19200`（可配置）
- 数据位：`8`
- 校验位：`无校验 + 1 停止位`
- 超时：`200ms`

### 数据寄存器（只读，功能码 0x03）

| 地址 | 名称 | 数据类型 | 范围 | 说明 |
|------|------|----------|------|------|
| `0x0000` | 单圈计数值 | uint16 | 0-分辨率 | 原始计数值，需换算成角度 |

**角度换算公式：**
```
角度(°) = (计数值 / 分辨率) × 360°
```

**示例：** 假设分辨率为 4096
```
读 0x0000 = 1024
角度 = (1024 / 4096) × 360° = 90°
```

### 控制寄存器（只写，功能码 0x06）

| 地址 | 名称 | 数据类型 | 范围 | 说明 |
|------|------|----------|------|------|
| `0x0008` | 置零命令 | uint16 | 1 | 写入 1 将当前位置设为零点 |
| `0x000B` | 设置当前值 | uint16 | 0-分辨率 | 直接设置当前计数值 |

### 使用示例

**读取角度：**
```
读 0x0000 = 2048
角度 = (2048 / 4096) × 360° = 180°
```

**设置零点：**
```
写 0x0008 = 1
```

**直接设置角度为 90°：**
```
计数值 = (90° / 360°) × 4096 = 1024
写 0x000B = 1024
```

| 地址 | 名称 | 数据类型 | 范围 | 说明 |

### 使用示例

**设置 CH1 电流为 0.5A：**

## 通信调度与轮询策略（修正版）

### 总线架构修正

**原设计问题：** 单总线挂 4 台设备，与前文"多总线工位模型"冲突，且不利于满足 AI1 事件检测时序。

**修正方案：** 支持多总线，推荐将 AQMD/编码器 与 DYN200 分开。

**推荐拓扑：**
```
上位机 (Master)
    |
    +-- RS485 总线 A (高优先级)
    |    |
    |    +-- AQMD (站号 0x01)
    |    +-- 编码器 (站号 0x03)
    |
    +-- RS485 总线 B (中优先级)
         |
         +-- DYN200 (站号 0x02)
         +-- 数控电源 (站号 0x04)
```

**优势：**
- 总线 A 专注于磁铁事件检测和角度读取，延迟更低
- 总线 B 处理扭矩/转速数据，互不干扰
- 单总线故障不影响另一总线

### 轮询周期与采样频率修正

**原设计问题：** 主轮询周期 50ms (20Hz)，但空载采样窗口要求 50Hz、2 秒 100 个样本，自相矛盾。

**修正方案：** 统一成设备可实现的采样策略。

**修正后的轮询策略：**

| 设备 | 轮询频率 | 周期 | 说明 |
|------|----------|------|------|
| AQMD AI1 | 30Hz | 33ms | 磁铁事件检测 |
| 编码器角度 | 30Hz | 33ms | 角度锁存 |
| DYN200 转速/扭矩 | 20Hz | 50ms | 受设备输出频率限制 |
| AQMD 电流 | 20Hz | 50ms | 空载采样 |
| 数控电源回读 | 10Hz | 100ms | 非关键路径 |

**空载采样窗口修正：**
- 采样时间：`2 秒`
- 采样频率：`20Hz`（与 AQMD 电流轮询频率一致）
- 样本数量：`40 个`（2s × 20Hz）
- 有效样本阈值：`≥ 32 个`（80%）

### AI1 去抖参数修正

**原设计问题：** 固定 50/50/100ms 去抖参数，未考虑轮询频率和轴速限制。

**修正方案：** 根据 30Hz 轮询频率和最大轴速调整参数。

**修正后的去抖参数：**

| 参数 | 值 | 说明 |
|------|-----|------|
| 最小高电平持续时间 | `66ms` | 2 个轮询周期 (33ms × 2) |
| 最小低电平持续时间 | `66ms` | 2 个轮询周期 |
| 事件屏蔽时间 | `200ms` | 防止同一磁铁重复触发 |
| 最大轴速限制 | `300 RPM` | 确保磁铁脉冲宽度 > 66ms |

**轴速与脉冲宽度关系：**
```
假设磁铁感应器宽度为 10mm，齿轮箱输出轴周长为 200mm
脉冲宽度 = (10mm / 200mm) / (转速 / 60s) = 3s / 转速

最大轴速 300 RPM 时：
脉冲宽度 = 3s / 300 = 10ms（仍需要磁铁感应器宽度足够）

实际应根据现场磁铁感应器和齿轮箱尺寸调整最大轴速限制。
```

## 修正总结

### P0 级别修正（阻塞实现）

1. ✅ **AQMD 寄存器映射**：修正为手册定义的地址和数据类型
   - `0x0010` = 实时 PWM（非运行状态）
   - `0x0040` = 有符号速度寄存器（非分离的正转/反转）
   - `0x0052/0x0053` = AI1/AI2 GPIO 电平（非 0x0012/0x0013）

2. ✅ **DYN200 寄存器映射**：修正为 32 位整数和正确的地址顺序
   - `0x00` = 扭矩（4 字节）
   - `0x02` = 转速（4 字节）
   - `0x04` = 功率（4 字节）

3. ✅ **编码器寄存器映射**：修正为原始计数值和正确的控制地址
   - `0x0000` = 单圈计数值（需换算）
   - `0x0008` = 置零命令
   - `0x000B` = 设置当前值

4. ✅ **数控电源寄存器映射**：修正为保持寄存器/输入寄存器/线圈三类
   - 保持寄存器 `0x0000-0x0003` = 电压/电流设定
   - 输入寄存器 `0x0000-0x0005` = 实际值回读
   - 线圈 `0x0000-0x0001` = 输出使能

### P1 级别修正（影响时序）

5. ✅ **轮询与采样频率统一**：
   - AQMD/编码器：30Hz
   - DYN200：20Hz（受设备限制）
   - 空载采样：20Hz × 2s = 40 个样本

6. ✅ **AI1 去抖参数联动**：
   - 根据 30Hz 轮询频率调整为 66ms
   - 增加最大轴速限制 300 RPM

7. ✅ **总线架构优化**：
   - 推荐双总线分离 AQMD/编码器 与 DYN200
   - 降低关键路径延迟
